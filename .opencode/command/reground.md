description: Re-anchor before a step — re-read the plan, state, and target files so the model isn't running on decayed memory
---

Do not trust implicit memory on a long task. Re-ground before continuing:

1. Re-read `@TASK.json` — restate the goal in one line, and the current step.
2. Re-read `@STATE.md` — what's built, what holds, what's a dead end.
3. Re-read the actual current contents of the file(s) this step touches (don't assume they
   match what you last saw — earlier edits or the reviewer may have changed them).
4. State the single next action, then do only that.

If `TASK.json`/`STATE.md` and the real files disagree, the files win — reconcile the ledger.
