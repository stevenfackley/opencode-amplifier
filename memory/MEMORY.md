# MEMORY.md — durable cross-session notes (index)

This file is loaded every session via the `instructions` array in `opencode.jsonc`. It is the
project's long-term memory: decisions, gotchas, and constraints that are NOT obvious from the
code and would otherwise be re-litigated every session.

One line per memory. Keep it short; put detail in the linked file under `memory/`.

## How to use
- **Read** this index at session start (it's auto-loaded).
- **Write** a new memory when you learn something durable: a non-obvious decision, a
  constraint, a gotcha that bit you, a "why we did it this way." Add a file under `memory/` and
  a one-line pointer here.
- Don't store what the code/git already records. Store the *why* and the *non-obvious*.
- For semantic recall across past sessions, enable the `episodic-memory` MCP (see
  `opencode.jsonc`) — it searches prior conversation transcripts.
